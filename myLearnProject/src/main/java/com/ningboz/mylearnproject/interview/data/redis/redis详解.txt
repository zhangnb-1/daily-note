 持久化：
    rdb: 生成快照
    aof: 日志追加 每条命令都同步到磁盘 || 固定时间内同步一次   aof可以重写覆盖之前的老日志，可以按照时间和日志大小来控制是否重写
    实际场景：混合使用

 三大经常经典场景
    穿透：访问不存在的key 永远都查不到，然后每次都去访问数据库
    雪崩：大量key同时到期，或者项目刚启动时redis预热没做好
    击穿：热门key过期的一瞬间大量并发

    穿透：参数校验、设置控制缓存、布隆过滤器
    雪崩、击穿：雪崩是大量key过期形成大量数据库访问，击穿是单个热点key的过期造成短期的大量数据库访问

延迟双删：
    先删缓存、再更新数据库、再删缓存

 淘汰策略:
    1. noeviction（不淘汰）
    2. volatile - lru（最近最少使用，针对设置了过期时间的键）
    3. allkeys - lru（最近最少使用，针对所有键）
    4. volatile - ttl（针对设置了过期时间的键，优先淘汰剩余存活时间短的键）
    5. volatile - random（随机淘汰设置了过期时间的键）
    6. allkeys - random（随机淘汰所有键）
