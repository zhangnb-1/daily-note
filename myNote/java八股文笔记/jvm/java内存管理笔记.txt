1.4   java堆
	jvm占用最大的一块内存，所有线程共享 
	几乎所有对象实例都在java堆中分配内存   
 	java堆是垃圾收集器管理的内存区域，垃圾收集器是GC垃圾回收算法的实现
	当 java 堆中没有内存可以进行实例分配，而且堆也无法再扩展的时候，java虚拟机会抛出OutOfMemory Error 即OOM: 内存溢出错误,
由此可见java堆的初始化是有配置的初始值和最大值的，当堆内存用完，且没有达到配置的最大值时，会扩展堆内存

2  虚拟机对象

 2.1  对象的创建 ： 指普通java对象，不包含数组和Class对象
	1. 类加载：把class加载到JVM运行时数据区的过程
	2. 检查加载：首先检查这个指令的参数是否能再常量池定位一个类的符号引用（此处含义不明朗），并且检查类是否已经被加载、解析和初始化过。
	3. 分配内存：在java堆中分配一块固定大小的内存
		3.1. 内存分配方式：
			3.1.1 指针碰撞：GC具有压缩整理功能，直接将指针向后移动固定长度即可，Seiral(JVM默认),ParNew 等GC都可以
			3.1.2 空闲列表：GC不具有压缩整理功能，出现零散的可分配内存块，比如CMS这种GC
		3.2  并发安全：
			3.2.1 cas机制：乐观锁
				
			3.2.2 分配缓冲（TLAB）：每个线程都在java堆中有单独分配内存的区域，互不干扰，本线程区域分配完了，再用cas机制分配新的缓冲区，是否使用TLAB可以用 -XX +/-UseTLAB参数来设定
	4. 内存空间初始化：
		内存分配完成后需要将分配的内存空间舒适化位零值，使用缓冲区TLAB的话，在分配TLAB时进行这一工作
	5.设置：对对象进行设置：比如属于哪个类的实例，类的元数据信息位置，对象的哈希码，对象的GC分代年龄等，都存放在对象的对象头中
	6.对象初始化：执行构造方法，初始化对象

2.2  对象的内存布局
	1. 对象头(Header) : 哈希码，GC分代年龄，锁状态标志，线程持有的锁，偏向线程ID，偏向时间戳
	2. 实例数据(Instance Data)：
		各种类型的字段内容，存储顺序收虚拟机分配策略参数（-XX FieldsAllocationStyle参数）的字段和源码中定义顺序的影响
	3. 对齐填充(Padding)：对象必须是8个字节的整数倍

2.3  对象的访问定位
	1. 句柄 ： java栈中的引用类型指向java堆中的句柄池的句柄地址，句柄再指向对象示例数据与类型数据各自的具体地址信息
	2. 直接指针：reference引用类型直接存储对象地址
	优劣对比 ：使用句柄时，每次垃圾收集器移动对象的时候只需要改变句柄的示例数据指针，reference不需要修改
		使用直接指针时，访问速度更快，节省了一次指针定位的时间开销，jvm默认的虚拟机HotSpot就是使用指针访问

2.4  对象存活的判断
	1. 引用计数法：在对象中添加一个引用计数器，因为存在对象相互引用的情况，需要额外机制来处理，会影响效率，故主流虚拟机没有采用

	2. 可达性分析（面试重点）：
		以GC Roots对象为起点，由这些起点向下搜索，搜索的路径为引用链，当一个对象到GC Roots 没有任何引用链相连时，此对象不可用
		2.1  主要GC roots 对象
			2.1.1 虚拟机栈（栈帧中的本地变量表）中引用的对象；各线程调用方法堆栈中使用到的参数、局部变量、临时变量	
			2.1.2 方法区中类静态属性引用的对象；java类的引用类型静态变量
			2.1.3 方法区中常量引用的对象；比如：字符串常量池里的引用
			2.1.4 本地方法站中JNI（Native方法）引用的对象，Native方法：java调用非java代码的接口，比如用java代码去调用c语言的代码
			2.1.5 JVM内部引用，被同步锁持有的对象，其他（这一行都非重点）
	
	3. Finalize 方法 
		当可达性分析判断不可达时，不会直接进行回收，会进行一次标记，当对象覆盖了Finalize 方法时，就可以在Finalize 方法中去拯救该对象（用处不大，执行较慢，不可靠，try-finally更好用）


2.5 对象的各种引用：
	1. 强引用: 
		举例：Object obj=new Object();
		任何情况下，只要强引用关联还在（与跟可达）,对象就不会被回收
	2. 软引用：
		举例：SoftReference<Object> sf = new SoftReference<Object>(obj);
			SoftReference类就是软引用类，通过get()方法回去软引用的对象
		在系统将要发生内存溢出的时候（OOM，OutOfMemory），软引用就会被回收
		使用场景：比如将一些图片读到内存，由于占有过大存储，将一些使用较少的图片设置为软引用，在内存溢出时直接回收，等需要的时候再从磁盘中读取就可
	3. 弱引用：WeakReference
		只要垃圾回收就直接回收，不管内存够不够，有没有可达性
