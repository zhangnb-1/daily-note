# Claude Code 最常用50个命令使用技巧

> 本文档整理了Claude Code最常用的50个命令，按使用频率/重要程度从高到低排序。

---

## 第一梯队：核心高频命令（日常必用）1-10

### 1. /plan - 规划命令 [最重要]
```
作用: 对代码修改进行规划，避免直接执行
使用场景:
  - 大规模重构前先确认方向
  - 不确定改动是否正确时
  - 需要先确认方向再执行
技巧:
  - 计划模式是安全的调试方式，可反复调整直到满意再执行
  - 先 plan 再执行，可以确保 Claude 理解你的意图
  - 使用 /plan 后可以随时调整计划内容
```

### 2. /help - 帮助命令
```
作用: 显示所有可用的 slash commands 和帮助信息
使用场景:
  - 忘记某个命令时快速查询
  - 了解新功能时使用
技巧:
  - 直接输入 /help 查看所有命令
  - 可配合具体问题使用
```

### 3. /clear - 清除上下文
```
作用: 清除当前对话的上下文，保留项目信息但清理会话历史
使用场景:
  - 切换到完全不相关的新任务时
  - 对话历史过长影响响应时
技巧:
  - 使用前可先保存重要信息到 CLAUDE.md
  - 不会清除项目配置和CLAUDE.md内容
```

### 4. /review - 代码审查
```
作用: 审查 Pull Request 的代码质量和安全性
使用场景:
  - PR Code Review
  - 合并前检查代码
  - 代码质量评估
技巧:
  - 配合具体 PR 信息使用，如 /review 后描述要审查的范围
  - 重点关注代码逻辑、安全漏洞、可读性
```

### 5. /simplify - 简化代码
```
作用: 审查并简化复杂代码
使用场景:
  - 代码可读性差时
  - 重构前简化逻辑
  - 优化嵌套过深的条件判断
技巧:
  - 一次简化一个文件或函数，避免大规模改动
  - 简化后务必检查是否保持原有功能
```

### 6. /commit - Git提交
```
作用: 创建 git 提交
使用场景:
  - 提交代码变更
  - 整理提交信息
技巧:
  - 使用 /commit <提交信息> 格式
  - 自动分析变更并生成规范的提交信息
  - 遵守团队 commit message 规范
```

### 7. /init - 初始化项目
```
作用: 初始化项目文档 (CLAUDE.md)
使用场景:
  - 新项目首次使用 Claude Code
  - 现有项目缺少文档时
技巧:
  - 文档越详细，Claude 理解项目越好
  - 建议包含：技术栈、目录结构、编码规范、常用命令
  - 已存在 CLAUDE.md 时会提示是否覆盖
```

### 8. /update-config - 配置管理
```
作用: 配置 Claude Code 的各种设置
使用场景:
  - 配置权限 (allow npm commands)
  - 设置环境变量
  - 修改权限策略
技巧:
  - 权限配置推荐使用项目级 .claude/settings.json
  - 便于团队共享配置
  - 常用配置: permissions、env、model、language
```

### 9. /security-review - 安全审查
```
作用: 对代码进行安全漏洞审查
使用场景:
  - 敏感功能（认证、支付等）代码上线前
  - 第三方依赖引入前审查
  - 安全合规检查
技巧:
  - 重点关注 SQL 注入、XSS、认证绕过等常见漏洞
  - 涉及用户数据或权限操作时务必使用
```

### 10. /loop - 循环任务
```
作用: 定期执行某个命令或任务
使用场景:
  - 监控部署状态
  - 定期检查构建结果
  - 轮询服务状态
技巧:
  - 默认 10 分钟间隔，可自定义如 /loop 5m /foo
  - 使用 <<autonomous-loop>> 可实现完全自主循环
  - 适合后台任务监控
```

## 第二梯队：效率提升命令 11-25

### 11. /claude-api - API开发辅助
```
作用: 辅助开发 Claude API / Anthropic SDK 应用
使用场景:
  - 构建 AI 应用时
  - 迁移 Claude API 版本
  - 优化 prompt caching
技巧:
  - 涉及 anthropic 或 @anthropic-ai/sdk 导入时自动触发
  - 会提示添加 prompt caching 以降低成本
```

### 12. /keybindings-help - 快捷键帮助
```
作用: 查看和自定义键盘快捷键
使用场景:
  - 想要修改快捷键
  - 添加自定义快捷键
  - 查看当前快捷键配置
技巧:
  - 支持 chord 绑定（组合键）
  - 可修改提交键、取消键、换行等
```

### 13. /less-permission-prompts - 权限优化
```
作用: 自动扫描常用只读操作，添加到白名单减少权限提示
使用场景:
  - 频繁遇到权限提示时
  - 设置新的开发环境时
技巧:
  - 主要针对 Bash(git *)、Read(/**) 等只读操作
  - 自动分析并添加常用操作到白名单
```

### 14. /branch - Git分支创建
```
作用: 创建 Git 分支
使用场景:
  - 创建新功能分支
  - 隔离开发任务
技巧:
  - 使用 /branch <分支名> 创建
  - 通常与 /commit 配合使用
```

### 15. -p, --print - 非交互式输出
```
作用: 非交互式输出（打印后退出）
使用场景:
  - 管道输入
  - 脚本集成
  - 自动化任务
技巧:
  - 常与管道命令结合使用
  - 输出后立即退出，不进入交互模式
```

### 16. -c, --continue - 继续对话
```
作用: 继续当前目录的最新对话
使用场景:
  - 恢复之前的会话
  - 继续中断的工作
技巧:
  - 快速恢复之前的上下文
  - 不需要指定会话 ID
```

### 17. -r, --resume - 恢复会话
```
作用: 恢复指定会话
使用场景:
  - 继续之前的工作
  - 精确控制会话
技巧:
  - 需要指定会话 UUID
  - 比 -c 更精确的会话控制
```

### 18. --model - 指定模型
```
作用: 指定使用的模型版本
使用场景:
  - 使用特定模型版本
  - 测试不同模型效果
技巧:
  - 可选: opus、sonnet、haiku
  - 不同模型有不同的能力和成本
```

### 19. --permission-mode - 权限模式
```
作用: 设置权限模式
使用场景:
  - 安全控制
  - 自动化脚本
技巧:
  - 可选: acceptEdits、auto、bypassPermissions、dontAsk、plan
  - plan 模式会先规划再执行，更安全
```

### 20. --debug - 调试模式
```
作用: 调试模式，排查问题
使用场景:
  - 排查问题
  - 查看详细日志
技巧:
  - 可添加 filter 过滤特定内容
  - 查看详细的执行日志
```

### 21. /agent - 启动子代理
```
作用: 启动子代理并行处理复杂任务
使用场景:
  - 并行处理多个独立任务
  - 复杂问题分解处理
技巧:
  - 可指定 agent 类型: general-purpose、Explore、Plan 等
  - 适合大规模代码探索和分析
```

### 22. /AskUserQuestion - 向用户提问
```
作用: 向用户提问收集需求
使用场景:
  - 收集需求
  - 确认选择
  - 多方案选择
技巧:
  - 支持多选项提问
  - 可用于复杂决策前的确认
```

### 23. /exit-plan-mode - 退出计划模式
```
作用: 退出计划模式
使用场景:
  - 取消当前计划
  - 直接开始执行
技巧:
  - 不保存当前计划
  - 与 /plan 配合使用
```

### 24. /task - 任务管理
```
作用: 创建和管理复杂的多步骤任务
使用场景:
  - 复杂项目分解
  - 大型任务跟踪
技巧:
  - 可创建任务列表
  - 跟踪任务进度
  - 支持任务依赖关系
```

### 25. /diff - 查看变更差异
```
作用: 查看变更差异
使用场景:
  - 对比代码变化
  - 审查修改内容
技巧:
  - 查看未提交的变更
  - 对比工作区和暂存区差异
```

## 使用技巧总结

1. **优先使用 /plan**: 对于不熟悉的代码修改，先用 /plan 确认方向再执行
2. **善用权限配置**: 使用 /update-config 或 /less-permission-prompts 减少打扰
3. **代码审查用 /review**: PR 合并前务必进行代码审查
4. **定期清理用 /clear**: 话题转换时清除旧上下文
5. **循环任务用 /loop**: 监控、轮询等场景使用
6. **安全审查用 /security-review**: 敏感代码上线前必查
7. **CLI 选项配合**: -p、-c、--debug 等选项可大幅提升效率
8. **会话管理**: 使用 --session-id、--fork-session 管理多会话
