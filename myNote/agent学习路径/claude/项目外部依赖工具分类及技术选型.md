# 项目外部依赖工具分类及技术选型

本文档对项目开发、部署、运维过程中涉及的外部依赖工具进行分类整理，涵盖各功能模块的技术选型选项。

---

## 一、开发阶段

### 1. 代码管理

| 工具 | 特点 | 适用场景 |
|------|------|----------|
| **Git** | 分布式版本控制，效率高，分支操作灵活 | 几乎所有项目 |
| **GitLab** | 自建Git仓库，支持CI/CD和项目管理 | 企业内部 |
| **GitHub** | 全球最大开源社区，生态丰富 | 开源项目 |
| **Gitea** | 轻量级自建方案，资源占用小 | 小团队 |
| **SVN** | 集中式版本控制，简单易用 | 传统企业项目 |

### 2. 依赖管理

| 工具 | 特点 | 适用场景 |
|------|------|----------|
| **Maven** | Java项目标准构建工具，依赖传递 | Java项目 |
| **Gradle** | 灵活性高，性能好，支持多语言 | Android/Java项目 |
| **npm** | Node.js包管理，生态最大 | 前端/Node.js项目 |
| **pip** | Python包管理 | Python项目 |
| **Go mod** | Go语言官方依赖管理 | Go项目 |

### 3. API文档

| 工具 | 特点 | 适用场景 |
|------|------|----------|
| **Swagger / OpenAPI** | 标准规范，代码优先 | REST API文档 |
| **SpringDoc** | Spring Boot集成方便 | Java Spring项目 |
| **Redoc** | 文档美观，支持OpenAPI 3.0 | 静态文档展示 |
| **Apifox** | API文档+测试+Mock一体化 | 团队协作 |

## 二、运行时基础设施（中间件）

### 4. 数据库

| 工具 | 特点 | 适用场景 |
|------|------|----------|
| **MySQL** | 开源关系型，生态成熟 | Web应用，中小规模 |
| **PostgreSQL** | 功能丰富，扩展性强 | 复杂查询，数据量大 |
| **Oracle** | 企业级，事务可靠 | 金融、政务 |
| **MongoDB** | 文档型，Schema灵活 | 日志、内容管理 |
| **TiDB** | 分布式NewSQL，MySQL兼容 | 高并发OLTP |

### 5. 缓存

| 工具 | 特点 | 适用场景 |
|------|------|----------|
| **Redis** | 内存KV，支持多种数据结构 | 高性能缓存，分布式锁 |
| **Memcached** | 简单高效，纯内存 | 简单缓存场景 |
| **Ehcache** | JVM内置，快速 | Java进程内缓存 |
| **Caffeine** | Java高性能缓存 | 本地缓存替代方案 |

### 6. 消息中间件

| 工具 | 特点 | 适用场景 |
|------|------|----------|
| **Kafka** | 高吞吐量，持久化，分布式 | 日志、大数据、流处理 |
| **RabbitMQ** | 功能丰富，支持多协议 | 企业集成，复杂路由 |
| **RocketMQ** | 阿里巴巴开源，事务消息 | 电商、金融场景 |
| **ActiveMQ** | 老牌消息队列 | 传统Java EE项目 |
| **Pulsar** | 云原生，存储计算分离 | 云原生架构 |

### 7. 搜索引擎

| 工具 | 特点 | 适用场景 |
|------|------|----------|
| **Elasticsearch** | 分布式全文搜索，生态完善 | 日志分析、全文检索 |
| **Solr** | Apache项目，成熟稳定 | 企业搜索 |
| **Meilisearch** | 轻量级，易部署 | 移动应用、中小型项目 |

### 8. 定时任务

| 工具 | 特点 | 适用场景 |
|------|------|----------|
| **xxl-job** | 分布式任务调度，轻量 | 互联网公司 |
| **Quartz** | Java标准，功能全面 | 单机或简单分布式 |
| **ElasticJob** | 分布式，弹性扩容 | 大规模任务 |
| **PowerJob** | 分布式，支持工作流 | 复杂调度场景 |

## 三、部署与容器化

### 9. 容器

| 工具 | 特点 | 适用场景 |
|------|------|----------|
| **Docker** | 事实标准，生态完善 | 容器化部署 |
| **Podman** | 无守护进程，安全性好 | 替代Docker |

### 10. 容器编排

| 工具 | 特点 | 适用场景 |
|------|------|----------|
| **Kubernetes (K8s)** | 事实标准，功能全面 | 生产环境 |
| **Docker Swarm** | 简单，Docker原生 | 小规模场景 |
| **Rancher** | K8s管理平台，简化运维 | 企业级K8s管理 |
| **K3s** | 轻量K8s，资源占用小 | 边缘计算、IoT |

### 11. CI/CD

| 工具 | 特点 | 适用场景 |
|------|------|----------|
| **Jenkins** | 插件丰富，可扩展 | 自建CI/CD |
| **GitLab CI** | GitLab集成，YAML配置 | GitLab用户 |
| **GitHub Actions** | GitHub原生，免费额度 | GitHub项目 |
| **Argo CD** | GitOps，声明式部署 | K8s云原生 |
| **Tekton** | K8s原生CI/CD | K8s环境 |

### 12. 配置管理与基础设施即代码

| 工具 | 特点 | 适用场景 |
|------|------|----------|
| **Ansible** | 无代理，SSH连接 | 配置管理、自动化 |
| **Terraform** | 声明式，跨平台 | 云资源编排 |
| **Pulumi** | 代码化IaC，支持多语言 | 复杂云架构 |
| **Chef / Puppet** | 传统配置管理工具 | 传统运维 |

## 技术选型建议

### 创业公司/小型项目
```
代码管理: GitHub
依赖管理: Maven/npm
数据库: MySQL
缓存: Redis
消息: RabbitMQ
定时任务: xxl-job
部署: Docker + Docker Compose
CI/CD: GitHub Actions
监控: Prometheus + Grafana
链路追踪: SkyWalking
日志: ELK
```

### 中大型互联网项目
```
代码管理: GitLab (自建)
依赖管理: Maven/Gradle
数据库: MySQL/TiDB + MongoDB
缓存: Redis Cluster
消息: Kafka
搜索引擎: Elasticsearch
定时任务: ElasticJob
部署: Kubernetes
CI/CD: Jenkins/GitLab CI + Argo CD
监控: Prometheus + Grafana
链路追踪: SkyWalking/Jaeger
日志: EFK
安全: Vault + Keycloak
```
