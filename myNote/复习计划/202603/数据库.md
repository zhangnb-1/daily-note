# MySQL 深度复习大纲 AI补充

## 1. 索引 internals
- B+ 树结构 vs B 树 vs Hash
- 聚簇索引 vs 二级索引 (回表)
- 覆盖索引 (Covering Index)
- 前缀索引与最左前缀原则
- 自适应哈希索引 (AHI)
- **索引下推 (ICP)**

## 2. 事务与隔离级别
- ACID 特性详解
- 隔离级别：RU, RC, RR, Serializable
- **MVCC (多版本并发控制)**: Read View, Undo Log 版本链
- 脏读、不可重复读、幻读的定义及解决

## 3. 锁机制 (Locking)
- 全局锁、表锁、行锁
- 共享锁 (S) vs 排他锁 (X)
- **意向锁 (IS/IX)** 的作用
- **间隙锁 (Gap Lock)** 与 临键锁 (Next-Key Lock)
- 死锁案例分析与排查 (`SHOW ENGINE INNODB STATUS`)

## 4. 日志系统 (Logging)
- **Redo Log** (WAL 技术，崩溃恢复，循环写)
- **Binlog** (归档，主从复制，追加写)
- **Undo Log** (MVCC，事务回滚)
- Two-Phase Commit (两阶段提交) 保证 Redo 与 Binlog 一致性

## 5. SQL 优化与执行计划
- `EXPLAIN` 深度解析 (type, key, rows, extra)
- `Using filesort` 与 `Using temporary` 优化
- 慢查询日志 (`slow_query_log`) 分析
- Profile 工具使用
- 常见反模式：`SELECT *`, `OR` 滥用，`LIKE '%...'`, 隐式类型转换

## 6. 架构演进：主从与分片
- **主从复制原理**：Binlog -> RelayLog, 异步/半同步/GTID
- **主从延迟**：原因与解决方案
- **分库分表策略**：垂直拆分 vs 水平拆分
- **分片后的挑战**：
    - 分布式 ID (Snowflake)
    - 跨库 Join 解决方案
    - 分布式事务 (Seata, TCC, 消息最终一致性)
    - 平滑扩容与数据迁移方案
    - 全局分页与排序优化

## 7. 内存管理与底层原理
- **Buffer Pool** 架构 (LRU 改进版，Young/Old 列表)
- 脏页刷新机制 (Flush)
- Change Buffer (写缓冲)
- Double Write Buffer (双写缓冲，防页断裂)

## 8. 实战场景题
- 深分页优化 (`limit 1000000, 10`)
- 大批量插入/删除优化
- 大表 DDL 操作 (Online DDL, pt-online-schema-change)
- CPU 100% / 磁盘 IO 飙高 排查思路
